# Contributing

Thanks for considering a contribution. This project is open source under the MIT license.

## How to contribute

1. Fork the repo
2. Create a feature branch (`git checkout -b my-feature`)
3. Make your changes
4. Run the starter template to verify nothing breaks (`cd starter && npm install && npm run dev`)
5. Commit with a clear message
6. Open a pull request

## What we welcome

- Playbook improvements (corrections, additional strategies, updated tooling)
- Starter template enhancements (accessibility, performance, new components)
- New templates (platform playbooks, content formats)
- Bug fixes
- Typo and clarity fixes

## What we don't merge

- Changes that add paid-only dependencies to the starter
- Marketing or promotional content
- Changes that break the "clone and run in 15 minutes" promise

## Style

- Keep playbook chapters practical and actionable
- Include specific examples, not abstract advice
- Code should be simple enough for someone learning Next.js to follow
- No em-dashes in prose. Use periods, commas, or restructure.

## Questions?

Open an issue. We'll get back to you.
